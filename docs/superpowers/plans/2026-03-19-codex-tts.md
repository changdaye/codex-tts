# Codex TTS Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a macOS-first `codex-tts` wrapper that launches interactive Codex, resolves the active rollout session, and speaks only the final assistant reply through a pluggable TTS backend.

**Architecture:** Use a Python CLI package with a thin wrapper entrypoint, a session resolver backed by `~/.codex/state_5.sqlite`, a rollout JSONL watcher that detects `phase:"final_answer"`, and a speech policy that emits one final utterance only. Keep terminal passthrough and TTS isolated behind small interfaces so the first release stays simple while future backends and richer policies remain possible.

**Tech Stack:** Python 3.11+, standard library (`argparse`, `json`, `pathlib`, `pty`, `selectors`, `sqlite3`, `subprocess`, `threading`, `tomllib`), `pytest`

## Implementation Notes

- Prefer built-in modules over external dependencies.
- Resolve the real `codex` binary with `shutil.which("codex")`, but reject a self-reference to `codex-tts`.
- Use polling for `state_5.sqlite` and rollout growth in V1 instead of file-system watchers.
- Keep final message extraction conservative: speak only `response_item.payload.phase == "final_answer"` assistant text.
- Treat missing session matches, parse failures, and TTS failures as non-fatal.

import json


def parse_rollout_line(line: str):
    row = json.loads(line)
    payload = row.get("payload", {})
    if (
        row.get("type") == "response_item"
        and payload.get("type") == "message"
        and payload.get("role") == "assistant"
        and payload.get("phase") == "final_answer"
    ):
        parts = payload.get("content", [])
        text = "".join(part.get("text", "") for part in parts if part.get("type") == "output_text")
        return ParsedRolloutEvent(kind="final_message", text=text)
    return ParsedRolloutEvent(kind="ignored", text="")
```

class SpeechPolicy:
    def __init__(self) -> None:
        self._seen: set[str] = set()

    def should_speak(self, event) -> bool:
        if event.kind != "final_message":
            return False
        text = event.text.strip()
        if not text or text in self._seen:
            return False
        self._seen.add(text)
        return True
```

import sqlite3


def resolve_active_thread(db_path, cwd: str, started_at: int, known_thread_ids: set[str]):
    conn = sqlite3.connect(db_path)
    rows = conn.execute(
        """
        select id, rollout_path, created_at, updated_at, cwd
        from threads
        where cwd = ?
        order by updated_at desc
        """,
        (cwd,),
    ).fetchall()
    for row in rows:
        if row[0] in known_thread_ids:
            continue
        if row[3] >= started_at or row[2] >= started_at:
            return ThreadRecord(thread_id=row[0], rollout_path=Path(row[1]), created_at=row[2], updated_at=row[3])
    return None
```

class SayBackend:
    def speak(self, text: str, *, voice: str, rate: int) -> None:
        run(["say", "-v", voice, "-r", str(rate), text], check=True)


def build_backend(name: str):
    if name == "say":

    raise ValueError(f"Unsupported backend: {name}")

def run_session(codex_cmd, config, state_db, home_dir) -> int:
    known = list_thread_ids(state_db)
    process = launch_codex(codex_cmd)
    thread = wait_for_active_thread(state_db, cwd=str(Path.cwd()), known_thread_ids=known)
    if thread is not None:
        final_text = wait_for_final_answer(thread.rollout_path)
        if final_text:
            backend = build_backend(config.backend)
            backend.speak(final_text, voice=config.voice, rate=config.rate)
    return process.wait()
```
